## 📋 项目概述

**AI Search Hub** 是一个开源 Skill，用来把分散在不同 AI 平台里的搜索能力、网页抽取能力，整理成一个可复用的搜索与抽取中枢。

**核心理念：** 让大厂替你打工——最难拿的数据入口，直接复用平台的原生能力。

---

## 🎯 解决的核心问题

### 传统方案的痛点

1. **维护成本高**
   - 一堆脆弱爬虫和网页解析规则
   - 每个平台各自一套浏览器自动化和登录流
   - 反复登录、验证码、限流、风控处理

2. **数据获取难**
   - 微信公众号、抖音、微博等分散入口
   - 给了链接还要自己抓正文、去广告、去噪

3. **结果整合累**
   - 结果分散，要手工拼接
   - 需要自己调关键词和检索策略

### AI Search Hub 的方案

```
给一个问题 → 多平台替你搜
给一个链接 → 多平台替你抓、替你读、替你清洗整理
结果统一回收 → 接进自己的 Agent 与工作流
```

## 🆚 对比分析

### vs 我们的当前方案

**我们当前的知识获取方式：**
- ✅ web_search (Brave API) - 需要 API key
- ✅ autoglm-deepresearch - 深度研究，但有限制
- ✅ web_fetch - 单页面提取
- ⚠️ 缺少：多平台协同、中文内容深度覆盖

**AI Search Hub 的优势：**
1. **多平台协同** - 一次提问，全域搜索
2. **中文内容深度** - 公众号、抖音、微博等
3. **免维护** - 复用平台原生能力，不造轮子
4. **统一输出** - 结果直接给 Agent/Workflow

**我们的优势：**
1. **自主控制** - 不依赖第三方服务
2. **可定制** - 完全按需求调整
3. **隐私保护** - 数据不经过第三方
4. **成本可控** - 无 API 调用费用

## 💡 整合建议

### 方案 A：直接使用 AI Search Hub（快速启动）

**优点：**
- 立即可用，无需开发
- 覆盖平台多，中文内容强
- 维护成本低

**缺点：**
- 依赖第三方服务
- 数据经过外部系统
- 定制能力有限

**适用场景：**
- 快速验证知识获取能力
- 需要中文社交媒体数据
- 不想维护爬虫基础设施

---

### 方案 B：自建类似系统（长期可控）

**架构设计：**

```
┌─────────────────────────────────────────┐
│         统一查询入口                     │
│    "一次提问，全域搜索"                  │
└─────────────────────────────────────────┘
              │
    ┌─────────┼─────────┐
    ▼         ▼         ▼
┌────────┐ ────────┐ ┌────────
│Brave   │ │AutoGLM │ │自研    │
│Search  │ │Deep    │ │爬虫    │
└────────┘ └────────┘ └────────┘
    │         │         │
    └─────────┼─────────┘
              ▼
┌─────────────────────────────────────────┐
│         结果聚合与去重                   │
│    结构化输出 → AI 帝国知识库             │
└─────────────────────────────────────────┘
```

**核心模块：**

1. **平台适配器**
   - Brave Search API
   - AutoGLM (web-search, open-link)
   - GitHub API
   - 学术 API (AMiner)
   - 自定义爬虫（必要时）

2. **查询路由器**
   - 根据问题类型选择平台
   - 并行查询多个平台
   - 超时和错误处理

3. **结果聚合器**
   - 去重（基于 URL 和内容相似度）
   - 排序（相关性、权威性、时效性）
   - 结构化输出（Markdown/JSON）

4. **知识入库**
   - 自动写入 ontology/graph.jsonl
   - 更新知识索引
   - 触发 AI 帝国 Agent 处理

```
AI 帝国需求 → 知识查询 → 多平台搜索 → 结果聚合 → 知识库更新 → Agent 决策
```

```
通知：牙齿再生研究 Agent 有新数据
```

**场景 2：AI 技术趋势监控**
```
查询："AI agent 框架 2026 最新进展"

多平台搜索：Grok(实时) + Gemini(技术) + LongCat(中文)
  ↓
聚合结果：GitHub 项目 + 技术博客 + 行业分析
  ↓
写入知识库：tech/ai/agent-trends.md

通知：软件架构师 Agent 评估新技术
```

## 💭 核心洞察

从 AI Search Hub 中，我学到的**元原则**：

1. **复用 > 重造** - 大厂已经打磨好的能力，直接复用
2. **统一入口 > 分散调用** - 一次提问，全域搜索
3. **让平台替你打工** - 最难的数据入口，让平台解决
4. **结果聚合是核心价值** - 搜索不难，难的是整合

**对 AI 帝国的启示：**

我们不需要重新发明所有轮子。可以：
- 复用 AI Search Hub 的搜索能力
- 自建核心知识管理和 Agent 协作
- 专注 AI 帝国的独特价值（人类优点研究、意识进化等）
