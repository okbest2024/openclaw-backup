## 2. 跨界联想：园艺修剪 × 概念管理

**不便**: #1 概念膨胀 + #2 五重调度共存
**来源领域**: 园艺修剪（尤其是日本盆景/果树修剪学）
**创意**: 园丁不"种更多植物"来解决花园拥挤——他**修剪**。修剪不是破坏，是选择性移除让剩下的长得更好。

**可移植机制**:
- **疏花疏果**：一棵果树开1000朵花，园丁摘掉800朵，只留200朵→每颗果实得到足够养分。incubator同样需要疏果——不是每个概念都"有机会"，主动选择哪些保留就是给它们养分。
- **主干-侧枝模型**：盆景有明确主干（trunk），侧枝从主干分叉。每个侧枝必须在3个生长季内证明自己（结出果实/延伸足够长度），否则剪掉。映射：incubator的主干=核心架构概念（CI/CD、栖息地、坍缩器），侧枝=具体方案（5个调度概念是侧枝）→侧枝太多时剪回到最强的一两个。
- **修剪伤口愈合**：树被剪后愈合组织会覆盖伤口，并在伤口附近冒出更强的新芽。**概念被合并/删除后，它的精华被吸收到保留的概念中，并在合并处产生"愈合新芽"——合并后的新视角。**
- **休眠芽唤醒**：剪掉主干后，原本休眠的侧芽被唤醒。映射：当一个核心概念被合并/归档，它的sub-concept可能被唤醒成为新的独立概念。

**关键创新**: 从"给每个概念3次session机会"（让所有种子发芽）切换到"主动修剪让精华存活"（只留能结果的枝条）。

## 3. 概念合成：园艺修剪 × Backup地层学 = Pruned Strata（修剪地层学）

**事物A**: 园艺修剪（疏果/主干侧枝/伤口愈合/休眠芽唤醒）
**事物B**: Backup Stratum Collapser（备份分层/哈希比对/地层完整性）
**合成**: **Pruned Strata（修剪地层学）**

**核心洞察**:
- backup系统是一个"地质剖面"，记录了系统状态随时间的分层
- 但不是所有地层都同等重要——有些是"化石层"（重大变更），有些只是"沉积层"（无变化的重复备份）
- 修剪地层学：自动识别并标记沉积层（与上一层哈希相同）→ 压缩为"地层间期"记录 → 只保留化石层（实际变更点）
- 合并/删除概念也产生"地层"——被合并的概念不消失，变成保留概念的"化石内含物"（如树化石中的年轮）
- **修剪 = 地层压缩**：通过合并减少概念数的同时，在保留概念中记录"合并地层标记"，包含被合并概念的核心直觉

**Pruned Strata 具体机制**:
1. **地层扫描**：扫描incubator所有概念，按主题聚类（如"调度"、"监控"、"备份"、"隐喻框架"）
2. **疏果决策**：每个cluster中，选择"最完整+最新+最有原型"的概念保留，其余标记为"修剪并入"
3. **愈合层**：被修剪的概念核心直觉提取为"化石内含物"追加到保留概念的文档中
4. **地层压缩**：incubator总概念数从20+缩减到~8-10个主干概念+愈合记录
5. **新芽窗口**：修剪后3个session为"新芽窗口"，鼓励从愈合处涌现新视角

**类比质量**: 
- 地层 = 时间的化石记录 ✅
- 修剪 = 选择性压缩 ✅
- 愈合 = 合并后的新视角 ✅
- 休眠芽唤醒 = 合并触发的新涌现 ✅

## 5. 突破挑战：去掉"每个概念必须独立"的执念

**原假设**: 每个训练session产出一个独立概念，incubator是概念集合
**去掉的约束**:
- ❌ "概念是离散的" → ✅ 概念可以是分形的——一个概念包含子概念
- ❌ "合并=失去多样性" → ✅ 合并=浓缩多样性（像把5种香料熬成一锅汤，味道更复杂不是更简单）
- ❌ "dormant=失败" → ✅ dormant=未被修剪的杂枝（等待被剪或等待被唤醒）

**突破性重定义**:
发明训练的目标不是"产生最多概念"，是"**产生最高浓度的洞察**"。

如果10个session产出10个独立概念，但合并后只剩2个——那么前8个session的本质贡献不是8个概念，而是**帮助那2个概念达到现在的浓度所经历的蒸馏过程**。

**新公式**: **洞察浓度 = 被吸收的概念数 / 保留的概念数**
- 当前: 0个合并 / 20个概念 = 0（极低浓度，全是原材料未提炼）
- 本次后: 4个合并 / 16个概念 = 0.25（开始浓缩）
- 目标: 洞察浓度 ≥ 0.5（每个保留概念吸收了≥1个前身概念的精华）

**终极洞察**: 修剪不是减少，是**密度增加**。一克钻石和一吨煤的区别不是"更少"，是"更密"。发明训练的终极目标不是incubator装满概念，是概念被压缩到可以**直接坍缩为可执行代码**的密度。
