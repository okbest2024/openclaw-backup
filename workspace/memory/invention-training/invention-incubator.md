## 概念 #9: 拱顶调度器 Vault Scheduler (Session 8) ⭐ NEW
**来源**: 大跨度无柱空间结构(机场航站楼) × 自清洁栖息地
**状态**: 概念提出，待生成最小原型 **[灌溉S11]** 核心重述：调度逻辑从各cron任务中解耦汇聚为中央拱顶。灌溉发现：拱顶在水文学视角下=河道的"地形"——不是控制水流，是决定水往哪流最省力。3session未渗透，但核心直觉（解耦+统一空间）仍有效。继续观察。
**描述**: 调度逻辑从各cron任务中解耦，汇聚为中央拱顶。任务只需"执行"和"发出信号"，拱顶根据全局信号密度自动调节间隔。高频成功路径聚合成承重肋(优先调度)，失败路径自然蒸散(自动降频→休眠→拆除)。不是给每个任务加信息素字段(柱子思维)，是建造一个统一承载的屋顶(无柱空间思维)。
**建筑师类比**：传统调度=每个任务是独立建筑(有自己屋顶、地基)。拱顶调度=所有任务在一个无柱大空间下自由流动(一个屋顶承载所有)。建筑师不控制人们怎么走路，他设计空间让最优路径自然浮现。
**与现有系统关系**:
- 信息素cron(概念#3) = 拱顶调度的前身/"每个柱子自己发出信号"
- 自清洁栖息地(概念#8) = 场地排水系统，拱顶调度 = 场地屋顶结构
- 发明栖息地(概念#7) = 整体场地，拱顶是其建筑学补完
**下一步**:
1. 生成scripts/vault-scheduler.js最小原型(30min)——最后机会，3个session未验证

## 概念 #11: Cron央行 / Cron自由市场 (Session 10) ⭐ NEW

**状态**: 概念提出，待生成最小原型
**描述**: 调度不是"决定什么时候执行什么"，是"消除执行摩擦"。三学科完整框架第三极：拱顶调度器(建筑学)×承载力管理器(生态学)×Cron央行(经济学)。核心机制：(1)利率走廊——全局基准cron间隔+成功率动态调节(>80%宽松移向floor, <30%紧缩移向ceiling, <10%加息休眠) (2)存款准备金——最低成功率门槛，连续失败率>阈值自动降频 (3)公开市场操作——高优任务购买紧急执行权 (4)宏观审慎——通道饱和度+会话池压力+token消耗联合监控。

**与现有系统关系**:

## 概念#17: 灾变年表 Catastrophe Chronology (Session 15) ⭐ NEW
**来源**: 庞贝层学 Pompeian Stratigraphy × 僵尸墓地 Cron Necropolis
**状态**: 概念提出，实验设计完成
**描述**: 每个cron尸体携带一份灾变时间线——从首次创建到最后死亡的所有状态变迁。不是snapshot快照，是timeline时间线。像庞贝城有7层灰烬（79年维苏威+62年地震等），cron的consecutiveErrors就是灰烬层数，每层代表一次灾变叠加。关键洞察：连续错误不是单事件，是叠加灾变序列。
**核心洞察**: S14尸检发现了死亡模式（同一模型栓塞/同一delivery断裂），但那是静态分类。灾变年表是动态分析——看一个任务如何从健康演变为病态再到死亡。Great Discovery Scientist从95s飙升到400s再死亡的退化曲线就是典型案例。
**地层学规则**:
- consecutiveErrors跃迁点 = 灾变节点（从0到1，从1到2，从2到3）

- lastRunStatus切换 = 状态灾变（ok-error是主震，error-ok是余震恢复）
- delivery=unknown = 通道灾变（消息无法送达=通讯中断）
**与现有系统关系**:

## 概念#18: 死亡牧场 Death Ranch (Session 15) ⭐ NEW
**来源**: 突破挑战（去掉必须复活的约束）× 庞贝城的2000年后重见天日隐喻
**状态**: 概念提出
**描述**: consecutiveErrors>=3的任务自动进入牧场（禁用+归档），每月放牧一次（自动检查是否条件改善）。牧场不是墓地——尸体不腐烂，而是放养，等待自然条件变化后可能回归。
**核心洞察**: 为什么叫牧场不叫墓地？因为牧场有回归的可能。庞贝城被灰烬掩埋后2000年才被重新发现——但发现后它就活了。consecutiveErrors=3是3的诅咒阈值：任务进入静默死亡，不再重试。与其无限重试消耗资源，不如主动禁用进入放养状态，等待外部条件变化（模型恢复/配额重置/通道修复）。
**牧场规则**:
- 入场条件: consecutiveErrors>=3 或 lastError包含timeout
- 自动禁用: 进入牧场的任务自动disabled=true
- 放牧周期: 每月1日自动检查牧场内所有任务（cron API调用验证）
- 回归条件: 如果对应模型/通道在其他任务中正常工作，尝试重新启用
- 永久安葬: 连续3个月放牧无改善，标记为permanent-disabled
**当前牧场候选**:
- Memory Training (consecutiveErrors=3)
- Great Discovery Scientist (consecutiveErrors=3)
- 思考共产主义 (consecutiveErrors=2，接近入场)
- 维度特性论训练两个 (consecutiveErrors=2，接近入场)
- body-repair (consecutiveErrors=21，已disabled)
- daily-ai-news (consecutiveErrors=6，已disabled)
**与现有系统关系**:
- 僵尸墓地(#16) = 分析死亡模式 → 死亡牧场 = 管理死亡（从诊断到处置）
- 自清洁栖息地(#8) = 退出机制 → 死亡牧场是其专业化版本（cron任务专用）
- 承载力管理器(#10) = 生态学视角 → 牧场释放承载力给健康任务
