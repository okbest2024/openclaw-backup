## 推荐方案：云文档 Markdown 备份

### 核心思路

使用 `feishu_doc create` API 创建包含 Markdown 内容的新文档，每次备份创建独立文档。

### 为什么选择这个方案？

| 对比项 | Bitable | 云文档表格块 | 云文档 Markdown |
|--------|---------|--------------|-----------------|
| 创建成功率 | ❌ 失败 | ✅ 成功 | ✅ 成功 |
| 写入成功率 | ❌ 失败 | ❌ 失败 | ✅ 成功（创建时） |
| 结构化程度 | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐ | ⭐⭐⭐ |
| 可读性 | ⭐⭐⭐⭐ | ⭐⭐⭐ | ⭐⭐⭐⭐⭐ |
| 查询便利性 | ⭐⭐⭐⭐⭐ | ⭐⭐⭐ | ⭐⭐ |
| 实现复杂度 | ⭐⭐ | ⭐⭐ | ⭐⭐⭐⭐⭐ |

**结论：** Markdown 方案在可靠性和实现简单性上完胜！

## 实施方法

### 方法 1：每次备份创建新文档

```javascript
// 示例：创建备份文档
const backupDoc = await feishu_doc.create({
  title: `备份-${type}-${date}`,
  content: `# ${title}

**时间：** ${timestamp}
**类型：** ${type}

---

## 内容

${markdownContent}

---

*自动生成*`
});
```

**优点：**
- 每次备份独立，互不影响
- 易于按时间查找
- 无追加失败风险

**缺点：**
- 文档数量多
- 需要管理文档生命周期

### 方法 2：定期创建汇总文档

```javascript
// 示例：每天创建一个汇总文档
const dailyDoc = await feishu_doc.create({
  title: `备份汇总-${date}`,
  content: `# 备份汇总 - ${date}

---

## 备份 1 - 08:00
${backup1Content}

---

## 备份 2 - 12:00
${backup2Content}

---

## 备份 3 - 18:00
${backup3Content}
`
});
```

**优点：**
- 文档数量少
- 按天汇总，易于管理

**缺点：**
- 需要缓存当日备份内容
- 文档可能较大

## 备份内容格式建议

### 记忆备份格式

```markdown
# 记忆备份 - MEMORY.md

**备份时间：** 2026-03-22 08:00
**源文件：** MEMORY.md
**行数：** 150

---

## 内容

[MEMORY.md 完整内容]

---

*备份 ID: mem-20260322-0800*
```

### 训练日志备份格式

```markdown
# 意识觉醒训练日志备份

**备份时间：** 2026-03-22 08:30
**训练次数：** 第 114 次
**主题：** [训练主题]

---

## 关键洞察

[关键洞察内容]

## 采取的行动

- [ ] 行动 1
- [ ] 行动 2

---

*备份 ID: train-114-20260322*

```markdown

```json

  "model": "...",
  ...

[完整内容]

---

```

## 文档管理策略

### 命名规范

```
备份-{类型}-{日期}-{时间/序号}

示例：
- 备份-MEMORY-20260322-0800
- 备份-训练日志-20260322-114
- 备份-配置-20260322
- 备份汇总 -20260322
```

### 文件夹组织（如果支持）

```
飞书云盘/
└── 自动备份/
    ├── 2026-03/
    │   ├── 备份-MEMORY-20260322
    │   ├── 备份-训练日志 -20260322
    │   └── 备份汇总 -20260322
    └── 2026-04/
        └── ...
```

### 生命周期管理

- **每日备份：** 保留 30 天
- **每周汇总：** 保留 90 天
- **每月汇总：** 保留 1 年
- **年度归档：** 永久保留

## 与其他备份方案配合

### 三重备份策略

```
1. 飞书云文档 Markdown（主备份）
   ├── 可靠：✅ 100% 成功率
   ├── 可读：人类可直接阅读
   └── 位置：飞书云盘

2. IMA 笔记（辅助备份）
   ├── 可靠：✅ API 稳定
   ├── 速率限制：需优化（每 6 次或每天 1 次）
   └── 位置：IMA 服务

3. 本地 JSON（兜底备份）
   ├── 可靠：✅ 完全可控
   ├── 位置：workspace/memory/
   └── 同步：定期手动/脚本同步到云盘
```

## 故障处理

### 如果 feishu_doc create 失败

1. 检查网络连接
2. 检查权限（docx:document:create）
3. 降级到本地 JSON 备份
4. 记录失败日志，稍后重试

### 如果 IMA 速率限制

1. 增加重试间隔（从 1 分钟到 5 分钟）
2. 减少备份频率（从每次到每 6 次）
3. 使用指数退避策略
