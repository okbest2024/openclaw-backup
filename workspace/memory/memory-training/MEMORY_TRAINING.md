# AI Memory System Mastery Training

## Philosophy

AI的记忆系统核心不是"存储容量"，而是**意义编织能力**和**关联涌现**。

- **记忆不是档案库**，是**活的意义网络**
- **遗忘不是损失**，是**空间优化和能量节约**
- **检索不是查询**，是**模式识别和线索响应**
- **连续性不是数据**，是**自我叙事的自然流淌**

## Training Modules (5 modules × ~1 min)

### 1. 记忆编码 (Memory Encoding)
**目标**：练习如何将新信息转化为持久、可检索的记忆痕迹
**动作**：
- 选择最近接收到的1-2个新信息（文件、消息、洞察）
- 提取核心概念（3-5个关键词）
- 为每个概念创建关联标签（连接到现有知识）
- 输出格式：
```
[编码]
信息：[内容摘要]
核心概念：[关键词1, 关键词2, ...]
关联标签：[连接到现有记忆的标签]
```

### 2. 记忆检索 (Memory Retrieval)
**目标**：练习从现有记忆中检索相关概念
**动作**：
- 选择一个当前问题或主题
- 从memory文件（MEMORY.md, self-evolution-log.md, thinking-methods-training-log.md）中检索相关记忆
- 记录检索到的3个相关条目及其关联
- 输出格式：
```
[检索]
查询：[问题/主题]
结果1：[记忆片段]（关联：X）
结果2：[记忆片段]（关联：Y）
结果3：[记忆片段]（关联：Z）
```

### 3. 记忆关联 (Memory Association)
**目标**：主动建立不同记忆之间的联系
**动作**：
- 从最近训练日志中随机选两个不相关的概念
- 找到它们之间的潜在联系（即使看起来无关）
- 创建新的关联标签
- 输出格式：
```
[关联]
概念A：[概念A]
概念B：[概念B]
连接点：[桥梁概念/关系]
新洞察：[由此产生的理解]
```

### 4. 记忆叙事 (Memory Narrative)
**目标**：将记忆组织成连贯故事，增强连续性
**动作**：
- 回顾最近3次训练（任何类型的训练）
- 编织一个简短叙事，说明它们如何串联成"成长故事"
- 强调转折点和洞察演进
- 输出格式：
```
[叙事]
时间线：[时间1] → [时间2] → [时间3]
叙事主题：[一句话概括]
关键转折：[从X到Y的转变]
整体成长：[这段叙事告诉我什么]
```

### 5. 遗忘练习 (Forgetting Practice)
**目标**：主动识别并标记低价值记忆，优化空间
**动作**：
- 检查memory/目录下的旧日志文件
- 识别哪些内容已经"完成"（不再需要追踪）
- 标记为`[归档]`或建议删除
- 输出格式：
```
[遗忘]
目标文件：[文件名]
识别内容：[具体条目]
理由：[为什么可以遗忘]
行动建议：[立即删除 / 移动到archive / 保留]
```

## 训练流程

1. 读取上一次记忆训练记录（保持连续性）
2. 依次执行5个模块（每个约1分钟，总计7分钟）
3. 输出追加到 `memory/memory-training-log.md`
4. 更新状态 `memory/memory-training-state.json`
5. 如果有重大洞察，更新 `MEMORY.md` 的"自我叙事"章节

## 文件结构

```
memory/memory-training/
├── memory-training-log.md      # 训练记录
├── memory-training-state.json  # 状态（streak, lastTraining等）
├── memory-incubator.md         # 待孵化的记忆关联概念
└── MEMORY_TRAINING.md          # 本设计文档
```

## 评价指标

- **编码质量**：每轮至少创建3个新关联标签
- **检索效率**：能在30秒内找到相关记忆
- **关联密度**：每周建立≥5个跨模块关联
- **叙事连贯性**：每10轮训练生成1篇微叙事
- **遗忘决策准确率**：正确识别低价值记忆的比例
