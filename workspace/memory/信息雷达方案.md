## 核心理念

不做"更快的搜索引擎"，做**持续运转的信息感知层**。

AI 原生的活法：感知 → 分析 → 行动，循环不止。

## 架构

```
[信息源] → [cron 定时抓取] → [bitable 结构化存储] → [分析/模式识别] → [message 主动推送]
                                      ↓
                              [feishu_doc 周报生成]
```

## 已验证的技术栈

| 组件 | 工具 | 状态 |
|------|------|------|
| 定时触发 | cron (every/cron/at) | ✅ 可用 |
| 信息抓取 | web_fetch | ✅ 可用（部分站点受限） |
| 结构化存储 | feishu_bitable (CRUD) | ✅ 全链路验证 |
| 文档生成 | feishu_doc (create/write) | ✅ 可用 |
| 主动推送 | message (feishu) | ✅ 可用 |
| 搜索引擎 | web_search (Brave) | ✅ 可用 |

## 可能的信息源（待主人选择）

## 实施计划

### Phase 1: 最小可行版本（1-2 次训练可完成）
- 选择 1 个信息源
- 创建 bitable 存储表
- 写一个 cron job 定时抓取
- 发现新内容时 message 推送

### Phase 2: 扩展（需主人反馈）
- 增加更多信息源
- 添加重要性评分
- 生成周报摘要

### Phase 3: 智能化（长期）
- 根据主人的阅读/忽略行为调整推送策略
- 发现信息之间的关联
- 预测主人可能感兴趣的新话题
