### 技术实现

**API：** `feishu_doc create`

**参数：**
```json
{
  "action": "create",
  "title": "{备份类型} 备份-{YYYYMMDD}-{HHmm}",
  "content": "# {标题}\n\n## 元数据\n...\n\n## 表格\n| 列 1 | 列 2 |...\n|---|---|...\n| 值 1 | 值 2 |..."
}
```

**命名规范：**
```

意识训练日志 -20260322-1530

```

**文档模板：**

## 🙋 常见问题

### Q1: 为什么 Bitable API 会失败？
**A:** 可能是 API 实现问题、参数格式问题或服务端限制。由于 API 返回的 400 错误没有详细消息，无法确定具体原因。建议联系飞书技术支持或等待 API 更新。

### Q2: 为什么不使用表格块方案？
**A:** 表格块方案需要多个 API 调用（create_table + write_table_cells），且 write_table_cells 返回 400 错误。相比之下，Markdown 表格方案只需一个 API 调用，更简单可靠。

### Q3: 每次创建新文档会不会产生大量文档？

### Q4: 如何实现增量备份？
**A:** 当前方案不支持增量备份。如需增量备份，建议使用备选方案 B（云盘 JSON 文件），或在应用层实现差异检测后创建新文档。

### Q5: 备份数据的安全性如何保障？
**A:** 飞书云文档的权限控制可以保障安全性。建议：
- 设置文档为私有（仅自己可见）
