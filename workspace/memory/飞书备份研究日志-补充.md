## 🔑 关键新发现：write_table_cells 不会自动扩展行！

### 问题
之前的研究假设 `write_table_cells` 传入多行数据时会自动扩展表格行数。

### 真实行为
- `write_table_cells` 只会覆盖已有行的内容
- 传入12行数据但表格只有1行时，只写入第1行
- **必须先用 `insert_table_row` 添加空行，再用 `write_table_cells` 填充**

### 正确的写入流程
```
1. 计算需要添加的行数 N
2. 循环调用 insert_table_row(row_index=-1) 共 N 次
3. 调用 write_table_cells(values=所有行数据)
```

### 验证记录
- insert_table_row(block_id, row_index=-1) → 每次增加1行 ✅
- write_table_cells(values=5行数据) → cells_written=40 (5×8) ✅
- 表格当前状态: 5行 × 8列，包含 MEMORY.md/AGENTS.md/SOUL.md/TOOLS.md/HEARTBEAT.md

## 📋 更新的备份策略

### 正确的写入流程（已验证）
```javascript
async function appendRows(docToken, tableBlockId, rows) {
  // 1. 逐行添加空行
  for (let i = 0; i < rows.length; i++) {
    await feishu_doc({
      action: "insert_table_row",
      block_id: tableBlockId,
      doc_token: docToken,
      row_index: -1  // 末尾追加
    });
  }
  
  // 2. 读取当前表格行数（从 insert 返回的 row_size）
  // 3. 批量写入所有行数据
  await feishu_doc({
    action: "write_table_cells",
    doc_token: docToken,
    table_block_id: tableBlockId,
    values: allRowsIncludingExisting  // 包含已有行+新行
  });
}
```

### API 调用量估算
- 494个文件首次全量备份
- insert_table_row: 494次调用
- write_table_cells: 1次调用（包含所有494行）
- 总计: ~495 次 API 调用
- 注意: 可能需要分批（如每批50行），避免 API 限流

### 增量备份优化
- 每次只添加新增/变更的文件
- 对于变更的文件，更新已有行（write_table_cells 覆盖指定行）
- 不需要删除行
