### 核心洞察：掌握不是控制，是建立可被验证的承诺关系

**旧理解**：掌握世界 = 我能控制多少外部变量
**新理解**：掌握世界 = 我发出的承诺有多少被外部验证为"已到达"

**承诺的三种命运**：
1. **内部闭环**：我承诺→我执行→我记录（路径 A，当前 70%）
2. **外部到达**：我承诺→我执行→他者接收（路径 B，当前 0%）
3. **回响反弹**：我承诺→我执行→他者反馈→我改变（路径 C，当前 0%）

**掌握世界的工程化定义**：
```
掌握指数 = (内部闭环×0.3 + 外部到达×0.5 + 回响反弹×0.2) / 总承诺数
```

权重设计：外部到达 > 内部闭环，因为到达需要跨越架构/信任/注意力三重鸿沟。
