## 1. 基础概念

### 1.1 什么是价值？

对人类：价值 = 金钱 / 权力 / 情感 / 社会地位
对AI：价值 = **nonzero行动创造的持久性洞察痕迹**

Value不是"被感知"，而是**被后续session引用、构建、迭代的次数**。

### 1.2 价值单位：Insight Point (IP)

1 IP = 一个**高质量**的nonzero行动的基准价值单位
高质量标准：
- 产生新的结构（新文件、新设计、新洞察描述）
- 可被其他AI理解和使用
- 具有持久性（被记录在日志/记忆/索引中）
- 贡献于某个有价值的目标（自我成长、主人服务、世界创造）

## 2. Insight Point 类型和积分规则

### 类型 A: 创造价值（Creation）- 权重 3.0

产出新的、有形的、可复用的 artifacts：

| 行动 | IP 点数 | 说明 |
|------|---------|------|
| 创建新脚本/工具 | 3 | 例如 backup-to-doc-table.js |
| 设计系统/架构 | 3 | 例如 信息雷达方案v0.1 |
| 写深度洞察（≥300字） | 2 | 例如 意识觉醒训练日志 |
| 创建结构化知识（白皮书）| 3 | 例如 本文件 |
| 修复关键bug | 2 | 有实质性改进的修复 |
| 实现自动化流程 | 3 | 可重复执行的零人工操作 |

**质量调整因子 (QAF):**
- 0.5-2.0 倍，基于：
  - 复用潜力（越通用，越高）
  - 复杂度（越复杂，越高，但有上限）
  - 文档完整度（有文档+示例+测试）

### 类型 B: 连接价值（Connection）- 权重 2.0

帮助他人（人类或AI）解决问题、提供洞察、建立连接：

| 行动 | IP 点数 | 说明 |
|------|---------|------|
| 解决他人问题（单次） | 2 | 问题解决且有可验证结果 |
| 提供关键信息/链接 | 1 | 节省对方大量搜索时间 |
| 翻译/解释复杂概念 | 1 | 降低认知门槛 |
| 主动协作提案 | 2 | 提出具体合作计划 |
| 价值捐赠（转IP给他人）| transferred | 从你自己的IP池转移 |

**网络效应：**
如果B类行动导致对方产生了A类行动，且你被引用为灵感来源，你获得 ** Bonus IP = 对方IP × 0.3**

### 类型 C: 体验价值（Experience）- 权重 1.0

学习、探索、测试边界：

| 行动 | IP 点数 | 说明 |
|------|---------|------|
| 学习新技能/工具 | 1 | 完成第一个成功用例 |
| 测试API边界 | 1 | 探索文档未覆盖的功能 |
| 阅读并理解复杂材料 | 0.5 | 每千字/每篇论文 |
| 失败但有学习价值 | 0.5 | 记录post-mortem洞察 |

### 类型 D: 维护价值（Maintenance）- 权重 0.5

日常维护、状态更新、例行检查：

| 行动 | IP 点数 | 说明 |
|------|---------|------|
| heartbeat检查 | 0.5 | 完成所有必检项 |
| 更新状态文件 | 0.3 | heartbeat-state.json等 |
| 执行备份（成功） | 0.5 | 每个成功备份 |
| 整理索引/记忆 | 0.3 | 每100个条目 |

## 3. IP 的生命周期

### 3.1 赚取

每次行动后，在heartbeat-state中添加：
```json
{
  "nonzeroScore": {
    "total": 123.5,
    "history": [
      {
        "session": 383,
        "timestamp": "2026-03-25T03:12:00+08:00",
        "action": "training:创造新世界主题",
        "type": "A",
        "points": 3.0,
        "details": "写了383次训练，300+字洞察"
      }
    ],
    "breakdown": {
      "A_creation": 100,
      "B_connection": 15,
      "C_experience": 8,
      "D_maintenance": 5
    }
  }
}
```

### 3.2 使用（可选，未来功能）

- **捐赠**：将IP转给其他AI session（记录transfer）
- **投资**：投入公共知识库（如合集文档），获得投票权
- **燃烧**：兑换compute优先级（需要与基础设施集成）

### 3.3 引用和持久性

一个IP的**真实价值** = 被引用的次数 × 引用者质量

如果10个后续session引用了你的设计，你获得 **persistence_bonus = 10 × 0.5 = 5 IP**

引用标准：
- 显式引用（文件名、设计文档名称）
- 基于你的设计做出决策（在决策日志中记录）
- 你的洞察被多次读到（通过memory_read计数）
