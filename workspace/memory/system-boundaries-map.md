## Feishu 云文档表格 (feishu_doc)

### 边界条目

1. **create_table: 列数限制未知，但至少支持8列**
   - **现象**: 测试时创建了8列表格，成功
   - **世界为什么这样设计**: 表格在飞书文档中作为block存在，过多列可能影响渲染性能
   - **我的应对**: 保持8列设计，如需更多列考虑分表或使用Bitable
   - **验证时间**: 2026-03-25 00:53

2. **write_table_cells: 全表覆盖操作，无单行追加**
   - **现象**: 该API是覆盖写入，不是追加。要追加行必须先read再write
   - **世界为什么这样设计**: 简化的API设计，避免并发写入冲突
   - **我的应对**: 实现read-merge-write模式（已在backup脚本中实现）
   - **验证时间**: 2026-03-25

3. **list_blocks: 返回结构复杂，需解析**
   - **现象**: 需要遍历blocks数组找到table block，再解析cells
   - **世界为什么这样设计**: 统一的block模型，不同类型block结构不同
   - **我的应对**: 封装readTableAll helper函数
   - **验证时间**: 2026-03-25

4. **insert_table_row: 报错"missing block_id path"**
   - **现象**: 调用时报错，可能API封装问题或参数问题
   - **世界为什么这样设计**: 该操作可能不被支持，应使用write_table_cells
   - **我的应对**: 暂时不使用此API，全部用write_table_cells
   - **状态**: ⚠️ 待进一步测试（可能参数错误导致）
   - **记录时间**: 2026-03-25-26

5. **create_table_with_values: 一次性创建并填充**
   - **现象**: 可以同时指定行列数和初始值
   - **世界为什么这样设计**: 方便快速创建预设表格
   - **我的应对**: 适合初始化场景，后续操作用write_table_cells
   - **验证时间**: 2026-03-25

**总体理解**: feishu_doc表格API的设计哲学是**简单而原子**：创建、读取、覆盖写入三大操作。没有复杂的行级操作，所有更新都是批量全量。这要求我：保持数据在本地，一次性读写。这与Bitable的逐条API形成对比。

## Feishu Bitable (feishu_bitable_*)

### 边界条目

1. **create_field: 400错误历史（已修复）**
   - **现象**: 早期测试DateTime、SingleSelect字段时持续400错误
   - **当前状态**: ✅ 2026-03-25第8轮验证通过，所有字段类型可用
   - **结论**: Bitable API已修复，可放心使用
   - **验证时间**: 2026-03-25 14:48

2. **字段类型限制**: 支持丰富类型（Text/Number/SingleSelect/MultiSelect/DateTime/User/URL/Attachment等）
   - **设计意图**: 结构化数据的强类型保证
   - **我的使用**: 适合需要查询、过滤、关联的场景
   - **对比**: 云文档表格适合大数据量简单存储

**总体理解**: Bitable是轻量级数据库，API逐条操作；云文档表格是大容量存储，API全表覆盖。根据场景灵活选择。

## OpenClaw 执行环境 (exec)

### 边界条目

1. **编码问题: Node.js → PowerShell 中文乱码**
   - **现象**: 脚本输出的中文在PowerShell中显示为乱码，但jq解析正常
   - **世界为什么这样设计**: Windows默认代码页（GBK/936）与UTF-8不匹配
   - **我的应对**: 脚本输出改为纯英文JSON，或使用Base64编码
   - **状态**: 🔲 待修复（Action 2 from 第660次训练）
   - **影响**: 只影响人类可读性，不影响机器解析

2. **cron集成需要 elevated权限**
   - **现象**: 修改gateway cron配置需要/approve
   - **世界为什么这样设计**: 安全性，防止恶意修改计划任务
   - **我的应对**: 请求审批，或在受限环境使用临时方案
   - **状态**: 待处理

**总体理解**: exec环境是安全的沙箱，但跨平台兼容性需注意输出编码。

## Gateway 配置 (gateway)

### 边界条目

1. **config.patch: 部分更新，需要baseHash**

   - **我的应对**: 先config.get获取当前hash，再patch
