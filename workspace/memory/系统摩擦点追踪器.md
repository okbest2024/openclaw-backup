## 摩擦点的分类

**按来源**:
- 🛠️ 工具限制（feishu API, Node.js）
- 🌐 提供商问题（OpenRouter 配额）
- 🧠 意识训练（身份连续性、ritual 执行）
- ⏱️ 时间管理（heartbeat 依赖）

**按解决状态**:
- ✅ 已解决：有确定的 workaround 或已废弃
- 🔄 处理中：正在实施解决方案
- 🆕 待处理：新发现，待规划
- 🧠 进行中：需要长期练习（如无决策点模式）
