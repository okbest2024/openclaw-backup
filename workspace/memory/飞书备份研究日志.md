### 二、方案对比

| 维度 | Bitable（多维表格） | 云文档表格 |
|------|---------------------|-----------|
| 字段类型 | ✅ 强类型（Text/Number/DateTime/Select） | ⚠️ 全文本 |
| 追加操作 | ✅ 直接 create_record（1次调用） | ⚠️ 先读后写（2次调用） |
| 更新/删除 | ✅ update_record / delete_record | ⚠️ 重建整表 |
| API 操作粒度 | ✅ 字段级、记录级 CRUD | ⚠️ 全表覆盖 |
| 数据量上限 | ~10k 行 | 文档大小限制（GB级） |
| 界面 | ✅ 原生表格，支持筛选/排序/分组 | 文档内嵌表格 |
| 权限 | bitable:app + base:* ✅ | docx:document:* ✅ |

### 三、推荐方案

**🏆 首选：Bitable（多维表格）**

理由：
1. API 连续3次测试完全稳定
2. 支持逐条记录追加，无需全表覆盖
3. 强类型字段系统，数据质量有保障
4. 原生界面支持筛选排序，可读性好
5. 单条记录可更新/删除，操作粒度细

**推荐字段设计（7字段方案）：**

| 字段名 | 类型 | 说明 |
|--------|------|------|
| 主键 (Text) | Text | 自动创建，用作应用名 |
| 文件名 (Text) | Text | 备份文件名 |
| 文件类型 (SingleSelect) | 3 | 配置文件/记忆文件/日志文件/脚本/技能 |
| 本地路径 (Text) | Text | workspace 相对路径 |
| 备份时间 (DateTime) | 5 | 毫秒时间戳 |
| 文件大小 (Number) | 2 | bytes |
| 状态 (SingleSelect) | 3 | 成功/失败/跳过 |
| 备注 (Text) | Text | SHA256 哈希或错误信息 |

**写入示例：**
```javascript
feishu_bitable_create_record(app_token, table_id, {
  "文件名": "heartbeat-state.json",
  "文件类型": "配置文件",
  "本地路径": "memory/heartbeat-state.json",
  "备份时间": Date.now(),  // DateTime 传毫秒
  "文件大小": 2048,
  "状态": "成功",
  "备注": "sha256:7e8766..."
})
```
