## 四、最终推荐：混合备份策略

### 🎯 推荐方案：Bitable + 云文档组合

**核心思路：** 用 Bitable 管理结构化索引，用云文档存储完整内容

**具体策略：**

1. **Bitable 作为索引库**
   - 表 1：备份索引表
     - 字段：备份 ID、备份类型、备份时间、文档链接、大小、状态
   - 表 2：训练记录表
     - 字段：训练序号、日期、主题、关键洞察、状态
   - 表 3：记忆条目表
     - 字段：记忆 ID、类别、摘要、创建时间、关联文档

2. **云文档作为内容库**
   - 每日备份：MEMORY.md 完整内容
   - 每周备份：SOUL.md、IDENTITY.md、AGENTS.md 等核心文件
   - 每月备份：完整 workspace 快照（JSON 格式）

3. **云盘作为归档库**
   - 按日期组织文件夹：`/backups/2026/03/2026-03-24/`
   - 存储原始 JSON 数据文件
   - 长期归档，不频繁访问

### 📋 实施建议

**日常备份（每次 session 结束时）：**
- 更新 Bitable 索引表（新增备份记录）
- 如有重要变更，创建云文档快照

**定期备份（heartbeat 触发，每 2-4 小时）：**
- 检查关键文件变化
- 如有变化，创建备份并更新索引

**完整备份（每日/每周）：**
- 导出完整 JSON 数据
- 创建云文档归档
- 更新 Bitable 索引

### 🔧 需要的额外权限

**当前权限已足够！** 无需额外申请。

### ⚠️ 注意事项

1. **速率限制：** 避免短时间内大量 API 调用，建议：
   - Bitable 记录创建：批量操作（一次 10-50 条）
   - 文档创建：限制在每小时 10-20 个以内

2. **数据一致性：** 
   - 先写内容，再更新索引
   - 失败时回滚索引记录

3. **清理策略：**
   - 保留最近 30 天的每日备份
   - 保留最近 12 周的周备份
   - 保留最近 12 个月的月备份
