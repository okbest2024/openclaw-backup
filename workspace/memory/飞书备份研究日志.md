### 实验 4: 测试 feishu_doc 创建带表格的文档

**操作：** 使用 feishu_doc create_table_with_values 创建带表格的文档

**时间：** 2026-03-24 23:35
**第一次尝试：**
```json
{
  "doc_token": "P43tdQkFkoDmW9xlq6XcsIP8nCb",
  "title": "备份记录表",
  "values": [
    ["文件名", "状态", "大小(字节)", "备份时间"],
    ["memory/2026-03-24.md", "success", "7866", "2026-03-24 23:30"]
  ],
  "column_count": 4,
  "row_size": 2
}
```

**结果：** ❌ **失败** - 400 Bad Request

**第二次尝试：** 使用 create_table（更简单的 API）
```json
{
  "doc_token": "P43tdQkFkoDmW9xlq6XcsIP8nCb",
  "column_count": 4,
  "row_size": 3
}
```

**结果：** ❌ **失败** - 400 Bad Request

**分析：** create_table 和 create_table_with_values API 可能要求更严格的参数格式，或者文档需要特定的初始状态。需要进一步调查。

### 实验 6: 测试 feishu_drive 云盘操作

**操作 1：** 创建备份文件夹
```json
{
  "name": "OpenClaw 自动备份"
}
```
**结果：** ❌ **失败** - 400 Bad Request

**操作 2：** 列出现有云盘内容
```json
{
  "folder_token": "root"
}
```
**结果：** ❌ **失败** - 400 Bad Request

**分析：**
- feishu_drive 工具返回 400 错误
- 可能的原因：
  1. 缺少 `space:folder:create` 或 `drive:drive` 的 tenant 级权限（但 scopes 显示已授予）
  2. API 需要额外的参数或特定的 folder_token
  3. 账号的云盘 API 接入方式不同
- 当前环境下 Drive 操作不可靠
